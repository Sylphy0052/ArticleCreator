# 反復とOK判定（固定）

## 反復フロー

1) YAML + md（記事/ソース）を受領
2) 表形式でスライド構成を提示
3) ユーザーが修正指示
4) 変更点サマリ → 更新後の表 を提示
5) OKまで繰り返す

## OKトリガー

以下のいずれかを含む入力を「確定」とみなす：

- OK
- 確定
- この構成で
- いきましょう

## 修正指示の例（ユーザー向けに提示してよい）

- 「Slide 3 を comparison_table にして、比較軸はコスト/効果/導入難易度」
- 「結論スライドを1枚目に移動して、以降を並べ替え」
- 「bulletsを減らして speaker_notes に寄せて」
- 「初心者向けに、専門用語は初出で一言定義」
- 「ビジュアルは写真ではなく図解寄りで」
