# Knowledge README（記事生成Gem・最小構成）

## このKnowledgeの役割

- 記事生成の「迷いどころ」（品質・表現・出典・安全）だけを固定し、出力のブレを減らす。

## 優先順位（矛盾がある場合）

1. ユーザー入力YAML（仕様）
2. 添付ファイル（Deep Research結果など）＋（任意）YAMLのsources（根拠）
3. このKnowledge
4. 一般知識（推測は明示）

## 重要ルール（最短）

- 不明点は断定しない：「不明」「要確認」「推測」を明記。
- 数値・比較・強い断定・固有名詞には可能な限り根拠（出典）を付ける。
- 記事の作りは「読者（audience）× レベル（level）× 意図（intent）」に合わせる。
